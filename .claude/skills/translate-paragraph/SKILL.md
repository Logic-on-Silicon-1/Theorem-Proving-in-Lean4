# /translate-paragraph

마크다운 파일을 받아 한국어 번역이 없는 영어 단락을 자동으로 찾아 번역합니다:
1. 한국어 번역 없는 영어 단락 자동 감지
2. 각 단락의 한국어 번역 생성
3. 번역 품질 검사 (의미 정확성, 자연성, 마크다운 보존, 전문 용어 일관성)
4. 문제 발견 시 자동 재번역

## 사용법

**단일 파일 번역** (한국어 없는 모든 단락 자동 번역):
```
/translate-paragraph "markdown/7. Inductive-Types.md"
```

**폴더 전체 번역** (폴더 내 모든 파일을 각각 새로운 대화에서 번역):
```
/translate-paragraph "markdown"
```

## 번역 규칙

- **전문 용어**: 영어 원문 유지 (예: API, Claude Code, type theory)
- **마크다운**: 코드블록, 링크, 강조 등 모두 보존
- **스타일**: 자연스러운 한국어, 학술적 톤 유지

## 검사 항목

✅ 의미 정확성 - 원문과 번역의 의미 동일 여부
✅ 자연스러운 표현 - 한국어의 자연스러움
✅ 마크다운 보존 - 코드블록, 링크 등 문법 유지
✅ 전문 용어 일관성 - 프로젝트 전반 용어 통일

검사 중 문제 발견 시 자동으로 재번역합니다.

## Claude의 작업

### 단일 파일 번역
사용자가 `/translate-paragraph "파일"` 명령을 사용하면:

1. **파일 읽기**: 마크다운 파일을 읽습니다
2. **단락 감지**: 한국어 번역이 없는 영어 단락들을 자동으로 찾습니다
   - 영어 단락 바로 아래에 한국어가 없는 경우 감지
   - 여러 단락이 있으면 순서대로 처리
3. **기존 번역 검사**: 한국어 번역이 이미 있는 단락을 발견하면:
   - 번역 규칙에 따라 품질 검사
   - 문제가 있으면 개선안 제시
4. **번역**: 각 단락을 번역 규칙에 따라 한국어로 번역합니다
5. **품질 검사**: 4가지 기준으로 번역을 검사합니다
6. **재번역**: 검사에서 문제 발견 시 (최대 2회까지) 자동으로 재번역합니다
7. **파일 수정**: 검증된 번역을 원문 단락 바로 아래에 추가합니다
8. **결과 보고**: 모든 번역 완료 및 검사 결과를 사용자에게 보고합니다

### 폴더 전체 번역 (병렬 처리)
사용자가 `/translate-paragraph "폴더"` 명령을 사용하면:

1. **폴더 검색**: 폴더 내의 모든 .md 파일을 찾습니다
2. **병렬 처리**: 모든 파일을 동시에 처리합니다
   - 여러 파일이 각각 새로운 대화에서 동시에 번역됨
   - 각 파일마다 독립적인 Claude 에이전트가 작동
   - 파일 개수만큼 동시 대화 생성
3. **동시 진행**: 각 파일의 번역은 병렬로 독립적으로 진행되며, 동일한 번역 규칙과 품질 검사를 적용합니다
4. **완료 보고**: 모든 파일의 번역이 완료되면 최종 결과를 보고합니다
